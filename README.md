# Консольное приложение для управления базой данных

**Описание**  
Проект реализует консольную базу данных: создание/удаление таблиц, вставка/чтение/обновление/удаление записей, с
хранением метаданных и данных в JSON-файлах. Реализовано полноценное CRUD-API через командную строку, с валидацией
типов, логированием, кешированием и подтверждением опасных операций.

Запись:  https://asciinema.org/connect/f027515f-8f09-43ab-9449-a247df48db60
## Возможности

- Создание и удаление таблиц (`create_table`, `drop_table`)
- Просмотр существующих таблиц (`list_tables`)
- Добавление, чтение, обновление и удаление записей (`insert`, `select`, `update`, `delete`)
- Вывод информации о таблице (`info`)
- Декораторы:
  - `@handle_db_errors` — обработка ошибок (`KeyError`, `ValueError`, `FileNotFoundError`)
  - `@confirm_action` — подтверждение опасных операций
  - `@log_time` — измерение времени выполнения функций
  - `create_cacher()` — кэширование результатов выборки
- Красивый вывод таблиц через `PrettyTable`
- Хранение данных таблиц и метаданных в JSON

## Основные команды

| Команда                                | Описание                               |
| -------------------------------------- | -------------------------------------- |
| `create_table <имя> <столбец:тип> ...` | Создать таблицу с указанными столбцами |
| `drop_table <имя>`                     | Удалить таблицу                        |
| `list_tables`                          | Показать список всех таблиц            |
| `help`                                 | Показать справку                       |
| `exit`                                 | Выйти из программы                     |


## CRUD-операции

| Команда                                                                | Описание                             |
| ---------------------------------------------------------------------- | ------------------------------------ |
| `insert into <таблица> values (<значения>)`                            | Добавить запись                      |
| `select from <таблица> [where <столбец>=<значение>]`                   | Получить записи (с фильтром или все) |
| `update <таблица> set <столбец>=<значение> where <столбец>=<значение>` | Обновить записи                      |
| `delete from <таблица> where <столбец>=<значение>`                     | Удалить записи                       |
| `info <таблица>`                                                       | Показать информацию о таблице        |


## Декораторы и кэширование
`@handle_db_errors` — автоматически перехватывает ошибки и выводит сообщения

`@confirm_action("название действия")` — спрашивает подтверждение пользователя перед удалением

`@log_time` — показывает время выполнения операций

`@create_cacher()` — позволяет кэшировать результаты select для ускорения повторных запросов
